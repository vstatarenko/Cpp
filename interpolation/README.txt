За основу интерполяции выбран метод интерполяции полиномом Лагранжа.
В зависимости от порядка(линейный, квадратичный, кубический) интерполяции,
задача сводится к нахождению соответствующих базисных многочленов Ларанжа.
За это отвечает метод

float Interpolation::CalcLagrangePolinom(float x)

При этом для каждого аргумента х нужно сначала найти близлежащие точки
	для линейной интерполяции - 2 точки
	квадратичной - 3 точки
	кубической -  4 точки
	и т.д.

После по этим близлежащим точкам и их значениям можно построить полином Лагранжа.


Замечания:
В каждом из методов я отдельно ищу близлежайшие точки. Этот процесс можно "завернуть"
в одну функцию(шаблон), но это можно отложить на будущее.
Также нет никаких перехватов ошибок (аргумент находится вне таблицы, количество точек не позволяют произвести интерполяцию),
кроме как текстового сообщения. Это тоже задача на будущее.
Нужно подумать об использовании метода Ньютона(и других), и насколько это целесообразно,
ведь интерполяция Лагранжа не предполагает сходимость. Это тема для исследования.
